## Logros de la semana

- Se reorganizó el repositorio del proyecto como **monorepo**, integrando backend y frontend.
- Se actualizó el **README principal**, agregando documentación detallada del módulo Cards.
- Se documentaron los **endpoints de tarjetas (Cards)** con sus métodos HTTP.
- Se describió el **modelo Card**, incluyendo campos, validaciones y comportamiento.
- Se agregó información sobre cómo **probar los endpoints usando Thunder Client / Postman**.
- Se verificó el correcto funcionamiento del backend de Cards mediante pruebas manuales.

---

## Bloqueos

- Dificultad inicial para identificar el backend correcto debido a la reorganización del repositorio.
- Integración completa entre frontend y backend aún no finalizada.
- Dependencia del flujo de autenticación JWT para pruebas completas desde frontend.

---

## Decisiones técnicas

- Centralizar la documentación técnica en el **README del monorepo**.
- Mantener el backend implementado con **FastAPI + SQLAlchemy**.
- Usar **JWT** para la autenticación y protección de endpoints.
- Realizar pruebas manuales iniciales con **Thunder Client / Postman**.
- Mantener el frontend separado en la carpeta `/frontend`.
